# EC2 Setup

## Overview
This phase covers launching EC2 instances in private subnets, configuring a bastion host for secure access, and setting up a web server.

---

## Instances Created
- 2 EC2 instances (Web Servers) in Private Subnets
- 1 Bastion Host in Public Subnet

---

## Bastion Host (Jump server)

### Purpose
- Acts as a secure entry point to access private EC2 instances

### Configuration
- Launched in Public Subnet
- Assigned Public IP
- Security Group:
  - SSH (22) (Allowed only from my IP)

---

## Private EC2 Instances

### Configuration
- 2 EC2 instances Deployed in Private Subnets
- No public IP assigned

### Security Group Rules
- SSH (22) > Allowed only from Bastion Security Group.
- HTTP (80) > Allowed (for Load Balancer Access)

---

## SSH Access Flow

### Step 1: Connect to Bastion Host 
- Connected to the Bastion with key
- Copied the .pem file from Windows to Bastion Server
- Changed permission of the .pem
```bash
chmod 400 key.pem
```

### Step 2: Connect to Private EC2 from Bastion
```bash
ssh -i key.pem ec2-user@<private-ip>
```

---

## Web Server Setup (httpd/Apache)

Run the following commands on both private EC2 instances:

```bash
sudo yum install httpd
sudo systemctl start httpd
sudo systemctl enable httpd
```
This will install, start and enable the service in the server.
(If the VM is not hosted on AWS then we have to add the service in firewall and reload it. To allow the traffic)

---

## Content COnfiguration

### Web Server 1 (text server)
```bash
echo "Hello from web server 1. The text server" | sudo tee /var/www/html/index.html
```

### Web Server 2 (Image server)
```bash
echo "Hello from web server 2. The image server" | sudo tee /var/www/html/index.html
cd /var/www/html/
wget <url of the image>
```
(This way we can download the setup the images in the server. Can change the server name short for easy access)

- Files:
  - index.html
  - a1.png
  - a2.png
